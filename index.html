A BRIEF HISTORY OF HTML

HTML was created by Tim Berners-Lee in 1991 to share documents over the World Wide Web. It has evolved through multiple versions, with HTML5 being the current standard.

HTML stands for HyperText Markup Language, and it's not a programming language - it's a markup language used to structure content on the web.

Structure Tags

<!DOCTYPE>
This tag tells the browser what version of HTML to expect and is placed at the very first line of any .html file to avoid rendering issues or unexpected behaviour. Note: This tag isn't an HTML element, it's a declaration and for this reason, it doesn't have a closing tag. For HTML5, we use this: <!DOCTYPE html>.
<html>
This defines the root element that wraps all HTML content. Any HTML code you write should go in this element. This element has an opening and a closing tag.
<head>
This tag contains the metadata, links to stylesheets, and the page title.
<body>
This holds all the visible content on a webpage, such as headings, paragraphs, images, and more.
<header>

<section>
The <section< element defines a section in a document. According to W3C's HTML5 documentation: "A section is a thematic grouping of content, typically with a heading." A Web site's home page could be split into sections for introduction, content, and contact information
<article>
The <article< element specifies independent, self-contained content. An article should make sense on its own, and it should be possible to read it independently from the rest of the web site. Examples of where an <article< element can be used:
<aside>
The <aside< element defines some content aside from the content it is placed in (like a sidebar). The aside content should be related to the surrounding content.
<footer>
The <footer< element specifies a footer for a document or section. A <footer< element should contain information about its containing element. A footer typically contains the author of the document, copyright information, links to terms of use, contact information, etc. You can have several <footer< elements in one document.
<div>
The <div> element is a block-level element that is often used as a container for other HTML elements. The <div> element has no required attributes, but style and class are common. When used together with CSS, the <div> element can be used to style blocks of content.

Form/Input Tags

<form>
HTML forms are used to collect user input. The <form> element defines an HTML form
<input>
The <input> element is the most important form element. The <input> element has many variations, depending on the type attribute.
<textarea>
The <textarea> element defines a multi-line input field (a text area).
<label>
The <label> tag defines a label for an <input> element. The <label> element does not render as anything special for the user. However, it provides a usability improvement for mouse users, because if the user clicks on the text within the <label> element, it toggles the control.
The for attribute of the <label> tag should be equal to the id attribute of the related element to bind them together.
<select>
The <select> element defines a drop-down list.
<optgroup>
The <optgroup> is used to group related options in a drop-down list. If you have a long list of options, groups of related options are easier to handle for a user.
<option>
The <option> elements defines the options to select. The list will normally show the first item as selected. You can add a selected attribute to define a predefined option.
<button>
The <button> element defines a clickable button.
